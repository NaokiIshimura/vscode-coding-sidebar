# 基本ルール
- mainブランチには直接コミットしないで
- 実装を始める前に、mainブランチから、`{バージョン}`のブランチを作成して
- package.jsonに記載されてるバージョンを`{バージョン}`にあげて

# 実装完了後
コードの修正が完了したらコンパイルして

# リリース準備
リリース準備を指示したら以下を実行して
- ブランチを作成して
- READMEを更新して
- README-JAを更新して
- CHANGELOGを更新して
- CHANGELOG-JAを更新して
- git commit
- pr 作成
  - pr title の先頭には「[{バージョン}] 」をつけて
- task.mdに変更があったらコミットして

# 言語
- UI上の表記は英語にして
- commit メッセージは英語にして
- pr title, descriptionは英語にして
